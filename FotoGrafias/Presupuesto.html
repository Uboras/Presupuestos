1. Definir el Alcance del Proyecto

Tiempo, Desarrollo y Detalle

Backend: (Servidor Principal)
Crear Los puntos de entrada al servidor (API).

- Desarrollo de la API.

Se crearan las entradas al servidor las cuales serán las direcciones electronicas a las cuales las sucursales usarán.

Se dividiran en 2:

     -Entradas de información ( Envió de formularios diarios y Carga de Mercaderias al sistema ).

     -Salida de información ( Reportes de stock a las sucursales/propios y Ajustes de stock )

Integración con una base de datos MySQL.

- Diseño de la base de datos: 2 días.

Se crea la base de datos y las tablas para almacenar la información que luego va a ser usarda para hacer los resportes

Se llenara con el el formulario el cual será enviado por las surcursales al servidor (Principal)

Implementación de autenticación y autorización.

Se Utilizará una palabra clave para el envio de la Información Diaria.

Se Utilizará una palabra clave para las Modificaciones de stock.

**ESTO SE ESTABLECE PARA TENER EL CONTROL DE LOS DATOS EN POCAS MANOS Y TENER EL CONTROL DE LA MANIPULACIÓN DEL PROGRAMA Y DATOS**

Documentación de la API.

- Documentación: 1 día.

- Para futuras Integraciónes de funciones

Frontend: (En cada Sucursal)
Dos interfaces de usuario (aplicaciones web o móviles).

Diseño y prototipado Desarrollo de la interfaz de usuario.

Interfaces:

     1. Para la surcursales que constara de reportes de stock, Ingresos de Mercaderias y Envio de formulario de ventas diarias.

     2. Para Usuario Con privilegios que otrogara reportes de ventas por sucursal, stock por sucursal, reportes Mensuales/anuales por Sucursal

Diseño responsivo.

Se diseñará para Computadora, tablet o celular

Otros desarrollos

- Configuración del entorno de desarrollo

- Pruebas de frontend y Backend

- Integración con el backend
